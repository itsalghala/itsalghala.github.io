HTB: Reactor Writeup

This writeup details the methodology used to compromise the Reactor machine, focusing on identifying vulnerabilities in modern web frameworks.

1. Reconnaissance

Initial scan using Nmap to identify open services:

# nmap -sC -sV -p- 10.129.10.200
22/tcp open ssh
3000/tcp open http (Next.js Application)

2. Vulnerability Research

Analysis of JavaScript files revealed the use of Next.js 15.0.3 and React 19.0.0, which are susceptible to prototype pollution leading to RCE.

3. Exploitation Path

Using a public Proof of Concept, I was able to verify the vulnerability and obtain a foothold as the node user.

4. Privilege Escalation

After lateral movement to the engineer account, I discovered a Node.js process running with --inspect enabled, allowing root access via the Chrome DevTools Protocol.
